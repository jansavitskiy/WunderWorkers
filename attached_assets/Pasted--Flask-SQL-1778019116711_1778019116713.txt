Контекст
У меня есть веб-приложение на Flask с админ-панелью и панелью сотрудника. Используется SQLite, SQLAlchemy, Jinja2. Уже реализованы: авторизация, роли (admin/employee), управление организациями, добавление отчётов сотрудниками, экспорт в CSV.

Что нужно добавить
1. Создать модель WorkType (Тип работы)
Поля: id, name (уникальное, строка), description (описание), is_active (булевый), created_at (дата)

Связь один-ко-многим с моделью Report (один тип работы — много отчётов)

2. Админ-панель для управления типами работ
Отдельная страница /admin/work_types (только для админа)

Форма добавления нового типа работы (название, описание)

Список всех типов работ с кнопками:

Редактировать (можно менять название и описание)

Активировать/Деактивировать (is_active)

Удалить (только если нет привязанных отчётов)

Кнопка в главном меню админа «Типы работ»

3. Форма сотрудника для добавления отчёта
Выпадающий список (select) с активными типами работ (только те, где is_active = True)

Обязательное поле — сотрудник должен выбрать тип работы

По умолчанию можно добавить несколько предустановленных типов: «Обычная работа», «Работа в выходные», «Сверхурочно», «Командировка»

4. Хранение в базе данных
В таблице reports добавить поле work_type_id (внешний ключ на work_types.id)

При сохранении отчёта обязательно сохранять выбранный тип работы

5. Отображение в админ-панели
В разделе «Отчёт за день» добавить колонку «Тип работы»

Показывать название типа работы для каждого отчёта

Возможность фильтрации отчётов по типу работы (выпадающий список)

6. Экспорт в CSV
В выгружаемом CSV добавить колонку «Тип работы» (после организации)

Значения должны быть текстовыми (название типа работы)

Все остальные колонки: Сотрудник, Организация, Тип работы, Часов, Что выполнено, Дата и время

Часы должны экспортироваться как числа (без кавычек), с точкой или запятой в зависимости от локали

7. Визуальное оформление
CSS для админ-панели: карточки с тенями, плавные анимации при наведении

Тёмная/светлая тема (переключатель, сохраняющий выбор в localStorage)

Для страницы отчётов: таблица с сортировкой по столбцам (клик на заголовок)

Фильтрация отчётов по дате (диапазон от и до)

Подсчёт итогового количества отчётов и суммы часов

Технические требования
Python 3.9+, Flask, SQLAlchemy, Flask-Migrate для миграций

WTForms для форм

Для работы с датами использовать UTC в БД, но отображать по московскому времени (MSK, UTC+3)

CSV с BOM (Byte Order Mark) для корректной кириллицы в Excel

Разделитель в CSV — точка с запятой (;)

Часы должны сохраняться как числа с плавающей точкой (поддержка 2.5 часа)

Пример ожидаемого CSV
csv
Сотрудник;Организация;Тип работы;Часов;Что выполнено;Дата и время
Иван Иванов;ООО Ромашка;Работа в выходные;5;Закончил отчёт;06.05.2026 14:30
Пётр Петров;ООО Берёзка;Обычная работа;4,5;Писал код;06.05.2026 13:30
Что должно получиться в итоге
Админ заходит в раздел «Типы работ», создаёт категории (например: «Работа в выходные», «Сверхурочно», «Ночная смена», «Больничный»)

Сотрудник при добавлении отчёта видит выпадающий список с этими категориями и выбирает нужную

В админ-панели в отчётах видна колонка «Тип работы»

При выгрузке CSV эта колонка тоже присутствует, и данные корректно открываются в Excel с русским текстом и числами

Пожалуйста, напиши полный код для реализации этой функциональности, включая:

Модель WorkType

Обновлённую модель Report

Маршруты для админ-панели (CRUD)

Обновлённую форму сотрудника

JS для интерактивности (подтверждение удаления, уведомления)

CSS для стилизации (с тёмной/светлой темой)

Обновлённый экспорт CSV с колонкой «Тип работы»

Весь код должен быть готов к использованию, с комментариями на русском языке.
