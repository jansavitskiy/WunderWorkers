Контекст
У меня есть веб‑приложение на Flask для учёта рабочего времени сотрудников. Уже работают: авторизация, роли (admin/employee), организации (с ИНН), типы работ, добавление отчётов сотрудниками (с выбором даты работы, часов, типа работы), экспорт CSV, админ‑панель с просмотром отчётов. Теперь нужно добавить расчёт стоимости отработанного времени в деньгах по индивидуальной ставке сотрудника, которую админ проставляет вручную (не коэффициенты, а конкретное число — например, 1250 рублей в час).

Цель
Админ должен иметь возможность вручную указать ставку (₽/час) для каждого сотрудника. Ставка одна для всех типов работ и дней недели. Если ставка не указана — в отчётах и CSV показывать 0 или прочерк. При добавлении отчёта система автоматически рассчитывает сумму к оплате (часы × ставка сотрудника) и сохраняет её в отчёт (или вычисляет на лету). В экспорте CSV появляется новая колонка "Сумма к оплате (руб)".

Что нужно реализовать
1. Модель User (добавить поле)
hourly_rate (Float, default=0.0) – ставка за час в рублях (например, 1250.00).

Поле может быть пустым (0 означает «ставка не задана»).

2. Админ-панель для управления ставками
На странице списка сотрудников (или отдельная вкладка «Ставки сотрудников»):

Таблица со столбцами: ФИО, Ник, Роль, Ставка (₽/час), Действия.

Напротив каждого сотрудника — поле ввода (числовое, с шагом 50-100) и кнопка «Сохранить».

Возможность массового обновления (опционально).

На странице просмотра отчётов у админа должна отображаться ставка сотрудника для информации.

3. Расчёт суммы к оплате
Логика: сумма = часы × hourly_rate_сотрудника

Если ставка не задана (0 или None): сумма = 0 (отображается как «Не задана» или 0).

Сумма округляется до двух десятичных знаков (рубли/копейки).

Добавить свойство total_amount в модель Report (как @property), которое вычисляет сумму на основе report.hours_worked и report.user.hourly_rate.

4. Сохранение в базу (опционально)
Сумму можно не сохранять в отдельном поле, а вычислять каждый раз при отображении (чтобы при изменении ставки все прошлые отчёты автоматически пересчитывались).

ИЛИ добавить поле calculated_amount в Report и пересчитывать при сохранении/обновлении ставки. Рекомендуется первый вариант (вычисление на лету) — проще и без денормализации.

5. UI для сотрудника
При заполнении отчёта сотрудник не видит свою ставку и сумму (только часы).

Можно добавить информационное сообщение: «Ваша ставка: X руб/час» (если задана) или «Ставка не задана, обратитесь к администратору» (опционально).

При вводе часов можно динамически показывать примерную сумму (AJAX-запрос к серверу) – опционально.

6. Админ-панель: отображение в отчётах
В разделе «Отчёт за день» (daily_report.html) добавить колонку «Сумма (₽)».

Отображать рассчитанную сумму для каждого отчёта.

Итоговая сумма за выбранный период (внизу таблицы) — сумма всех total_amount.

7. Экспорт в CSV
Добавить колонку «Сумма к оплате (руб)» после колонки «Часов» (или перед описанием).

Формат: число с двумя десятичными знаками (например, 1250.00 или 1250,00).

Для Excel – использовать запятую как разделитель дробной части (.replace('.', ',')).

Если ставка не задана – выводить 0 или пустую строку.

8. JS на стороне клиента (опционально)
В форме сотрудника при выборе часов отправлять AJAX-запрос на /get_calculated_amount с параметрами hours_worked и получать примерную сумму.

Технические требования
Использовать существующие модели: User (добавить hourly_rate), Report (не менять структуру, только добавить свойство total_amount).

Миграции БД: добавить поле hourly_rate в таблицу users.

Все вычисления производить на стороне бэкенда (Python), в свойстве модели Report.total_amount.

Для отображения в шаблонах использовать фильтр форматирования: {{ report.total_amount|round(2) }}.

В CSV выводить число с разделителем-запятой и точностью 2 знака.

Пример ожидаемого CSV
csv
Сотрудник;Организация;ИНН;Тип работы;Часов;Сумма к оплате (руб);Что выполнено;Дата работы;Дата создания
Иван Иванов;ООО Ромашка;7701234567;Обычная работа;8;10000,00;Писал код;05.05.2026;07.05.2026 14:30
Пётр Петров;ООО Берёзка;771234567890;Выходной день;5;6250,00;Закончил отчёт;03.05.2026;07.05.2026 15:00
(При ставке Ивана 1250₽/час → 8×1250=10000; ставке Петра 1250₽/час → 5×1250=6250)

Что должно получиться в итоге
Админ заходит в раздел «Сотрудники» и вручную вводит ставку для каждого (например, 1250₽/час, 1500₽/час).

Сотрудник заполняет отчёт (часы, тип работы) – сумма рассчитывается автоматически (часы × ставка).

Админ в отчётах видит колонку «Сумма к оплате» (она же выгружается в CSV).

CSV-файл содержит все данные + сумму.

При изменении ставки сотрудника все прошлые отчёты автоматически пересчитываются (если сумма вычисляется динамически).

Шаги для реализации (ожидаемый код)

Обновить модель User – добавить атрибут hourly_rate.

Обновить модель Report – добавить свойство @property total_amount:

python
@property
def total_amount(self):
    rate = self.user.hourly_rate if self.user else 0.0
    return round(self.hours_worked * rate, 2)
